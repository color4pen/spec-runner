# Spec: Resume Context Auto Injection

## Requirements

### Requirement: The executor shall generate resume context from state

When an agent step is executed as the first step of a resume run and the job has
a `resumePoint`, the executor SHALL build deterministic resume context from
`JobState` and pass it through the existing `resumePrompt` field.

#### Scenario: plain resume after escalation

**Given** a job state with `resumePoint.step` equal to the step being executed
**And** `state.steps[resumePoint.step]` contains one or more prior attempts
**And** the resume command supplied no human prompt
**When** `StepExecutor` builds the `AgentRunContext`
**Then** `ctx.session.resumePrompt` contains an auto-generated context block
**And** the context includes the current attempt count for that step
**And** the context includes the previous attempt verdict
**And** the context includes the `resumePoint.reason`
**And** the context explains that artifacts already present in the worktree may
come from an earlier attempt and do not prove the current attempt is complete.

### Requirement: Human resume prose shall supplement automatic context

Human-provided resume prose from `--prompt` or `/resume` comments MUST be
preserved as an additional supplement after the automatically generated context.

#### Scenario: resume with human prompt

**Given** a job state that qualifies for automatic resume context
**And** `deps.resumePrompt` contains human prose
**When** `StepExecutor` builds the `AgentRunContext`
**Then** `ctx.session.resumePrompt` contains the automatic context
**And** `ctx.session.resumePrompt` contains the human prose
**And** the human prose appears after the automatic context.

### Requirement: Non-resume runs shall not receive resume context

The executor MUST NOT inject automatic resume context for normal first-run
execution or for state that has no active `resumePoint`.

#### Scenario: initial run

**Given** a running job state with no `resumePoint`
**And** no human resume prompt
**When** `StepExecutor` builds the `AgentRunContext`
**Then** `ctx.session.resumePrompt` is `undefined`.

### Requirement: Automatic context shall be deterministic

The automatic resume context MUST be generated by pure formatting of stored
state fields. It MUST NOT call an LLM, infer unstored facts, inspect the
worktree, or depend on wall-clock time.

#### Scenario: same state produces same text

**Given** the same `JobState`, step name, and human prompt input
**When** the automatic context builder is called twice
**Then** both calls return identical strings.

### Requirement: Future state-backed context sections shall have an extension point

The implementation SHALL expose a narrow builder structure where additional
state-backed sections can be appended later, including decision ledger data,
without changing adapter code or the `AgentRunContext` contract.

#### Scenario: future decision ledger support

**Given** a future implementation needs to include decision records from state
**When** it extends resume context generation
**Then** it can add a new deterministic section to the builder
**And** it does not need to change local, managed, or codex adapter injection.
